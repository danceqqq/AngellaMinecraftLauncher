# Инструкция по настройке лаунчера

## Быстрый старт

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка GitHub репозитория

Откройте файл `main.js` и найдите строку:
```javascript
const GITHUB_REPO = 'YOUR_GITHUB_USERNAME/YOUR_REPO';
```

Замените на ваш репозиторий. Например:
```javascript
const GITHUB_REPO = 'myusername/minecraft-hardcore-server';
```

### 3. Настройка GitHub Releases для авто-обновлений

1. Создайте репозиторий на GitHub
2. Перейдите в раздел **Releases**
3. Создайте новый релиз (например, v1.0.0)
4. Загрузите файл `dwa.jar` как прикрепленный файл (asset)
5. Лаунчер будет автоматически проверять последний релиз

**Важно:** Файл должен называться `dwa.jar` или быть первым файлом в списке assets релиза.

### 4. Запуск лаунчера

```bash
npm start
```

## Настройка пути к Minecraft

По умолчанию лаунчер использует путь:
```
C:\Users\MyAngel\AppData\Roaming\.minecraft\versions\dwa\dwa.jar
```

Если ваш путь отличается, измените константы в начале файла `main.js`:
```javascript
const MINECRAFT_PATH = path.join(process.env.APPDATA, '.minecraft');
const VERSION_PATH = path.join(MINECRAFT_PATH, 'versions', 'dwa');
const JAR_FILE = path.join(VERSION_PATH, 'dwa.jar');
```

## Настройка параметров запуска

В файле `main.js` в функции `launch-game` можно изменить:
- `-Xmx2G` - максимальный объем памяти (например, `-Xmx4G` для 4GB)
- `-Xms1G` - начальный объем памяти

## Проверка работы

1. Запустите лаунчер: `npm start`
2. Создайте профиль с вашим никнеймом
3. Нажмите "Запустить игру"
4. Если возникнут ошибки, проверьте:
   - Установлена ли Java
   - Существует ли файл `dwa.jar` по указанному пути
   - Правильно ли настроен путь к Minecraft

## Сборка установщика

Для создания установочного файла:
```bash
npm run build-win
```

Установщик будет в папке `dist/`.

## Решение проблем

### Java не найдена
- Убедитесь, что Java установлена
- Проверьте, что Java добавлена в PATH
- Или укажите полный путь к Java в функции `findJavaPath()`

### Ошибка при запуске игры
- Проверьте путь к `dwa.jar`
- Убедитесь, что файл существует
- Проверьте права доступа к файлу

### Ошибка авто-обновления
- Проверьте правильность указания GitHub репозитория
- Убедитесь, что репозиторий публичный или у вас есть доступ
- Проверьте интернет-соединение
